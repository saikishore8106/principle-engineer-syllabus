1

Foundations — Core Depth & LLD Mastery

Weeks 1–4 · Solidify the base that architecture decisions are built on

Week 01
Low-Level Design (LLD) & Design Patterns
SOLID principles — deep dives with real refactoring exercises on legacy code
Design patterns: all 23 GoF — Creational (Factory, Builder, Singleton), Structural (Adapter, Decorator, Proxy), Behavioral (Observer, Strategy, Command)
OOP vs Functional tradeoffs for backend systems — when to use which
Practice problems: Parking Lot, Library Management System, ATM Machine, Chess Board
UML class diagrams, sequence diagrams — write them fluently
LLD Design any object-oriented system in 30 mins
Week 02
Data Structures & Algorithm Complexity
Advanced trees: B-Tree, B+ Tree, LSM Tree, Trie, Segment Tree — real-world use contexts
Graphs: BFS/DFS, Dijkstra, Bellman-Ford, topological sort in system design contexts
Consistent hashing, bloom filters, skip lists, HyperLogLog
Amortized complexity, space-time tradeoffs at scale
When to use which structure — interview-level decision-making
DSA Explain complexity tradeoffs to any audience
Week 03
Database Design — Deep Dive
Normalization 1NF–5NF, denormalization strategies for read-heavy systems
Indexing: B-Tree, Hash, Composite, Partial, Covering, GIN/GiST indexes
ACID vs BASE, isolation levels (Read Committed, Serializable), MVCC internals
Query optimization, EXPLAIN plans, N+1 problem, connection pooling
Schema design for e-commerce, SaaS multi-tenancy, time-series, audit logs
PostgreSQL internals: WAL, vacuum, autovacuum tuning
DB ERD design + query plan review proficiency
Week 04
Concurrency, OS & Networking Depth
Threads vs processes, coroutines, async I/O — epoll, io_uring, event loops
Locks, mutexes, semaphores, deadlock detection, prevention & recovery
TCP/IP internals, HTTP/2, HTTP/3 (QUIC), gRPC, WebSockets — when to use what
TLS/mTLS handshake, DNS internals, latency anatomy (RTT, TTFB)
Practice: Build a concurrent rate limiter (token bucket + sliding window) from scratch
SYS Confident in OS-level performance discussions
2

System Architecture — HLD & Distributed Systems

Weeks 5–8 · Think in systems, design for scale, reason about failure

Week 05
High-Level Design (HLD) Framework
Capacity estimation: QPS, storage, bandwidth, memory — back-of-napkin math fluency
Load balancing: L4 vs L7, algorithms (round-robin, least-conn, consistent hashing rings)
CDN design, DNS-based routing, anycast, GeoDNS
API Gateway patterns, rate limiting, circuit breaker, bulkhead
Design: URL shortener (bit.ly), Pastebin, Dropbox, Google Drive at scale
HLD 45-min design interview simulation — pass rate
Week 06
Distributed Systems & Consensus
CAP theorem, PACELC model, partition tolerance in practice
Raft consensus algorithm (deep), Paxos concepts, leader election patterns
Vector clocks, CRDTs, last-write-wins, eventual consistency patterns
2-Phase Commit, Saga pattern (choreography vs orchestration), distributed transactions
Design: distributed lock service, leader election, global ID generator (Snowflake)
Paper reading: Amazon Dynamo, Google Spanner, Raft paper
DIST Deep dive into distributed systems papers
Week 07
Microservices, Messaging & Event-Driven
Microservices vs monolith vs modular monolith — decision framework with real tradeoffs
Kafka internals: partitions, offsets, consumer groups, ISR, replication, compaction
Event sourcing, CQRS, outbox pattern, transactional inbox
Service mesh: Istio/Envoy, sidecar pattern, circuit breakers, retries, timeouts
Design: Notification system (push/SMS/email), ride-sharing real-time backend
MSG Kafka + event-driven architecture design
Week 08
Caching, Search & Storage Systems
Cache topologies: cache-aside, write-through, write-back, refresh-ahead — when to use each
Redis internals: data structures, persistence (RDB/AOF), cluster mode, Lua scripting
Elasticsearch: inverted index internals, sharding, relevance tuning, aggregations
NoSQL: Cassandra data modeling (partition key design), DynamoDB access patterns
Design: Twitter timeline (fan-out on write vs read), YouTube video storage pipeline
CACHE Multi-tier caching architecture design

AIOps, Platform Engineering & Principal Mindset

Weeks 9–12 · Lead technical direction, AI-native systems, engineering excellence

Week 09

LLM integration patterns: RAG architecture, vector databases (Pinecone, Weaviate, pgvector)
Observability AI: intelligent alerting, auto-remediation, predictive scaling

Cloud-Native & Platform Architecture
Kubernetes architecture: etcd, scheduler, kubelet, controllers, operators, CRDs
Platform engineering: Internal Developer Platforms (IDP), Golden paths, Backstage
Multi-region active-active deployments, chaos engineering (Chaos Monkey)
Cost engineering: FinOps principles, rightsizing, spot instances strategy, reserved capacity
Security architecture: Zero Trust, SPIFFE/SPIRE, secrets management (Vault), supply chain
CLOUD K8s + platform design whiteboard review
Week 11
Reliability, SRE & Performance Engineering
SLO / SLI / Error budget discipline — how to set them, own them, and fight for them
Distributed tracing: OpenTelemetry, Jaeger, correlation IDs, trace sampling
Performance profiling: CPU flamegraphs (perf, async-profiler), memory heap analysis
Incident management: blameless post-mortems, runbook automation, toil reduction
Load testing at scale: k6, Locust, capacity planning spreadsheets
SRE Design a 99.99% uptime architecture with runbooks
Week 12
Principal Eng Skills + Capstone Project
Architecture Decision Records (ADRs), RFC writing, technical proposal documents
Tech radar, make vs buy vs adopt decisions, vendor evaluation frameworks
Influencing without authority, stakeholder alignment, technical roadmapping

Daily Study Schedule — 6–8 Hours Structure

Repeatable daily rhythm for maximum retention and skill building

Morning Block — Deep Theory (3 hrs)

07:00 – 08:30
Concept Study
Read + take notes on that week's core topic. Books, papers, official docs.
08:30 – 09:30
Video Deep-dive
MIT OCW, DDIA chapters, conference talks (re:Invent, QCon, StrangeLoop)
09:30 – 10:00
Active Recall
Write summary from memory. Anki flashcards. No peeking at notes.

Afternoon Block — Hands-on Practice (3 hrs)

11:00 – 12:30
Design Exercise
Whiteboard system design. Time-boxed 45 min, then self-critique vs model answer.
12:30 – 13:00
Lunch + Walk
No screens. Let concepts consolidate. Physical reset is non-negotiable.
14:00 – 15:30
Coding / Labs
Implement a component: rate limiter, cache, Kafka consumer, Redis cluster locally.

Evening Block — Review + Output (2 hrs)

16:00 – 17:00
Mock Interview
Leetcode system design, or Pramp peer session (2–3× per week). Record yourself.
17:00 – 18:00
Write + Publish
Blog post, ADR, or design doc. Writing output is what separates principal engineers.
18:00 – 18:30
Weekly Retro (Fri)
What clicked? What didn't? Adjust next week's pacing accordingly.
